# 📋 코드읽기 (CodeReadr) — 프로젝트 계획서

## 🎯 프로젝트 정의

### 한 줄 요약
웹사이트 코드를 전혀 모르는 사람이 자기 프로젝트 파일(또는 MD 분석 문서)을 업로드하면, AI가 6단계로 나누어 코드 이해 → 기술 스택 학습 → 코드 리뷰 → 프롬프트 작성까지 가르쳐주는 학습 앱

### 왜 만드는가?
Cursor, Claude Code, Bolt, Lovable 같은 AI 도구 덕분에 코드를 모르는 사람도 웹사이트를 만들 수 있게 됐다. 하지만 "만들 수는 있는데 읽을 수는 없는" 사람이 폭발적으로 늘고 있다. 기존 코딩 교육(노마드코더, Codecademy 등)은 범용 예제로 가르치기 때문에, 자기 프로젝트 코드는 여전히 이해하지 못한다. 이 앱은 **내 프로젝트 코드를 교재로** 써서, 실제로 코드를 읽고 리뷰하고 더 나은 프롬프트를 쓸 수 있게 만들어주는 것이 목표다.

### 누가 쓰는가?
- **1순위:** AI 도구(vibe coding)로 웹사이트를 만들었지만, 코드를 설명 못 하는 사람
- **2순위:** AI로 포트폴리오를 만들었는데 면접에서 "이 코드 설명해보세요"가 두려운 취준생
- **3순위:** 외주로 받은 코드를 이해하고 싶은 비개발 창업자

### 최종 학습 목표
코드를 몰랐던 사람이:
1. 프로젝트 전체 구조를 설명할 수 있고
2. 사용된 기술 스택을 이해하고
3. 코드 흐름을 따라 읽을 수 있고
4. 코드를 자기 말로 설명할 수 있고
5. 코드의 문제점을 찾아 리뷰할 수 있고
6. **구체적인 프롬프트를 작성해서 AI와 협업하며 웹사이트를 만들어나갈 수 있다**

## 🏗️ MVP 범위 (v1.0)

### 포함하는 것
- 단일 유저 (로그인 없음, 내가 혼자 테스트)
- 파일 업로드 (.zip, .md, 개별 코드 파일)
- Gemini 무료 API로 코드 분석
- 6단계 학습 시스템 전체
- localStorage에 학습 진행률 저장
- 다크 모드 기본 UI

### 포함하지 않는 것 (나중에 추가)
- 회원가입 / 로그인
- GitHub URL 자동 분석 (UI만 배치, 기능은 "준비 중")
- 다중 유저 지원
- 학습 데이터 DB 저장
- 결제 / 구독 시스템
- SNS 공유 카드 ("나는 87% 이해합니다")
- 모바일 최적화 (데스크탑 우선)

## 📚 6단계 학습 시스템 상세

### 1단계: 프로젝트 스캔 (입문) 🗺️

**목적:** "이 프로젝트가 뭔지" 전체 그림을 잡는다

**작동 방식:**
1. 유저가 파일 업로드
2. AI가 전체 파일을 읽고 분석
3. 프로젝트 지도(폴더 트리 시각화) 생성
4. 각 파일의 역할을 한 줄로 요약
5. 프로젝트의 핵심 기능 3~5가지 식별

**화면 구성:**
- 왼쪽: 시각적 폴더 트리 (클릭하면 파일 설명 표시)
- 오른쪽: 프로젝트 요약 카드
  - "이 프로젝트는 [Next.js]로 만든 [커플 앱]입니다"
  - "총 [23]개 파일, [3]가지 핵심 기능"
  - 핵심 기능 목록 (예: 데이트 기록, 스티커, 편지)

**AI 프롬프트 핵심:**
```
이 프로젝트의 모든 파일을 분석해서 다음을 JSON으로 알려줘:
1. 프로젝트가 무엇인지 비개발자도 이해할 수 있는 한 줄 설명
2. 사용된 프레임워크/라이브러리
3. 핵심 기능 3~5가지
4. 각 파일의 역할 한 줄 요약
전문 용어를 쓸 때는 반드시 괄호 안에 쉬운 설명을 추가해줘.
```

**완료 조건:** 유저가 "프로젝트 지도"를 보고 전체 구조를 이해했다고 체크

---

### 2단계: 기술 스택 학습 (기초) 🧱

**목적:** "이 프로젝트에 쓰인 기술들이 뭔지" 하나씩 배운다

**작동 방식:**
1. AI가 프로젝트에서 사용된 기술 스택 식별
2. 각 기술을 카드 형태로 표시
3. 카드를 클릭하면 해당 기술의 쉬운 설명 + 프로젝트에서 어디에 쓰였는지 표시
4. 비유를 활용한 설명 (예: "Supabase는 데이터를 저장하는 창고 같은 것")

**화면 구성:**
- 기술 스택 카드 그리드
- 각 카드: 기술명 + 아이콘 + 한 줄 설명 + "이 프로젝트에서의 역할"
- 카드 클릭 → 상세 설명 패널 열림
  - 이 기술이 뭔지 (비유로 설명)
  - 이 프로젝트에서 어디에 쓰였는지 (파일명 + 코드 스니펫)
  - 왜 이 기술을 선택했는지

**AI 프롬프트 핵심:**
```
이 프로젝트에서 사용된 모든 기술/프레임워크/라이브러리를 식별하고,
각각에 대해 다음을 JSON으로 알려줘:
1. 기술명
2. 코딩을 전혀 모르는 사람도 이해할 수 있는 비유 설명 (3문장 이내)
3. 이 프로젝트에서 구체적으로 어디에 사용되었는지 (파일명과 줄 번호)
4. 이 기술이 없으면 어떤 문제가 생기는지
```

**완료 조건:** 모든 기술 스택 카드를 열어보고 체크

---

### 3단계: 코드 따라읽기 (중급) 🔍

**목적:** "유저가 버튼을 누르면 코드에서 뭐가 일어나는지" 흐름을 따라간다

**작동 방식:**
1. AI가 프로젝트의 주요 유저 행동(user flow)을 식별
2. 각 행동별로 코드 실행 순서를 추적
3. 순서대로 해당 코드 블록을 보여주며 한 줄씩 설명
4. "이 줄에서 → 저 파일로 이동" 같은 파일 간 연결도 시각화

**화면 구성:**
- 상단: 유저 행동 선택 드롭다운 ("로그인할 때", "데이터 저장할 때" 등)
- 메인: 단계별 코드 워크스루
  - Step 1: `page.tsx` 37번 줄 → "여기서 폼 데이터를 수집해요"
  - Step 2: `useDateRecords.ts` 12번 줄 → "이 함수가 데이터를 서버로 보내요"
  - Step 3: Supabase → "이 코드가 실제로 데이터베이스에 저장하는 부분이에요"
- 각 코드 블록 옆에 한국어 해설 말풍선

**AI 프롬프트 핵심:**
```
이 프로젝트에서 유저가 할 수 있는 주요 행동 5가지를 식별하고,
각 행동에 대해 코드가 실행되는 순서를 추적해줘.
각 단계마다 다음을 JSON으로 알려줘:
1. 실행되는 파일명과 함수명
2. 해당 코드 블록 (시작줄~끝줄)
3. 이 코드가 하는 일을 비개발자도 이해할 수 있게 설명
4. 다음 단계로 넘어가는 이유 (이 코드가 저 파일의 저 함수를 호출하기 때문에...)
```

**완료 조건:** 최소 3개 유저 행동의 코드 흐름을 따라읽고 체크

---

### 4단계: 퀴즈 & 설명 연습 (심화) 📝

**목적:** "이 코드를 자기 말로 설명할 수 있는지" 확인한다

**작동 방식:**
1. AI가 프로젝트 코드에서 문제 출제
2. 코드 블록을 보여주고 "이 함수가 하는 일을 설명해보세요" 질문
3. 유저가 텍스트로 답변 작성
4. AI가 답변을 평가하고 부족한 부분 피드백
5. 점수 + 모범 답변 제공

**문제 유형:**
- "이 함수가 하는 일을 설명해보세요" (서술형)
- "이 코드의 실행 결과는?" (객관식)
- "이 변수의 역할은?" (단답형)
- "이 컴포넌트가 화면에 뭘 보여주나요?" (서술형)

**화면 구성:**
- 코드 블록 + 질문
- 답변 입력 영역
- "제출" 버튼 → AI 평가 결과
  - 점수 (0~100)
  - 잘한 점 / 부족한 점
  - 모범 답변

**완료 조건:** 평균 70점 이상 달성

---

### 5단계: 코드 리뷰 훈련 (최종) 🔎

**목적:** "이 코드의 문제점을 찾을 수 있는지" 훈련한다

**작동 방식:**
1. AI가 실제 프로젝트 코드에서 개선 가능한 부분 식별
2. 코드 블록을 보여주고 "이 코드에서 개선할 점을 찾아보세요" 출제
3. 힌트 시스템 (1단계: 영역 힌트, 2단계: 구체적 힌트, 3단계: 답 공개)
4. AI 평가 + 개선된 코드 예시 제공

**리뷰 카테고리:**
- 보안 취약점 (예: API 키 노출, XSS 위험)
- 성능 이슈 (예: 불필요한 리렌더링, 무한 루프 위험)
- 코드 구조 (예: 파일이 너무 긴, 반복 코드)
- 에러 처리 (예: try-catch 누락)
- 접근성/UX (예: alt 태그 누락)

**화면 구성:**
- 코드 블록 + "이 코드에서 문제를 찾아보세요"
- 힌트 버튼 (3단계)
- 리뷰 의견 입력 영역
- AI 평가 + 모범 리뷰 + 개선 코드 비교 (before/after)

**완료 조건:** 5개 이상 리뷰 챌린지 완료

---

### 6단계: 프롬프트 마스터 (실전) 🚀

**목적:** "AI에게 구체적으로 지시해서 기능을 추가할 수 있는지" 훈련한다

**작동 방식:**
1. AI가 시나리오 제시: "이 프로젝트에 [사진 첨부 기능]을 추가하고 싶어요"
2. 유저가 Claude CLI에게 지시할 프롬프트를 작성
3. AI가 프롬프트를 평가:
   - 구체성 (어떤 파일을 수정해야 하는지 언급했는가?)
   - 기술 용어 (정확한 용어를 사용했는가?)
   - 완성도 (빠뜨린 단계가 없는가?)
4. before/after 비교: 학습 전 프롬프트 vs 학습 후 프롬프트

**시나리오 예시:**
- "데이트 기록에 사진 첨부 기능 추가"
  - 나쁜 프롬프트: "사진 넣는 거 만들어줘"
  - 좋은 프롬프트: "page.tsx의 폼에 이미지 업로드 input을 추가하고, Supabase Storage에 저장한 후, date_records 테이블에 image_url 컬럼을 추가해서 연결해줘. 이미지 미리보기도 보여줘."
- "다크모드 토글 기능 추가"
- "데이터 검색 기능 추가"
- "알림 기능 추가"

**화면 구성:**
- 시나리오 카드 (기능 추가 요청 설명)
- 프롬프트 작성 에디터 (문법 하이라이팅)
- "평가받기" 버튼 → AI 피드백
  - 점수 (0~100)
  - 구체성 / 기술 용어 / 완성도 각각 평가
  - "이렇게 쓰면 더 좋아요" 개선된 프롬프트 예시
- before/after 비교 섹션

**완료 조건:** 3개 이상 시나리오에서 80점 이상 달성

## 📌 개발 시 참고사항

### 작업 환경
- **에디터:** Cursor (무료 플랜, 에디터 기능만 사용)
- **AI 코딩:** Claude CLI (Cursor 내장 터미널에서 실행)
- **터미널:** Cursor 내장 터미널 (PowerShell)
- **버전관리:** Git + GitHub

### Claude CLI 작업 플로우
```
1. Cursor에서 프로젝트 폴더 열기
2. 내장 터미널에서 claude 실행
3. 자연어로 코드 생성/수정 지시
4. Cursor 에디터에서 변경된 코드 확인
5. 코드를 읽고 이해한 후 다음 지시
```

### Gemini API 키 발급
1. [aistudio.google.com](https://aistudio.google.com) 접속
2. Google 계정으로 로그인
3. "Get API Key" 클릭
4. API 키 복사
5. `.env.local` 파일에 `NEXT_PUBLIC_GEMINI_API_KEY=복사한키` 저장
